# 03 — Memory：讓 Claude 記住你

## 你會學到什麼

- Memory 和 CLAUDE.md 的差異與配合方式
- Memory 的儲存位置和生命週期
- 如何主動讓 Claude 記住特定資訊
- 查看、更新、刪除 memory
- 什麼適合放 Memory、什麼適合放 CLAUDE.md

預估時間：15 分鐘

---

## 核心概念

### Memory 是什麼

Memory 是 Claude 自動記錄的個人筆記，儲存在你本機的 `~/.claude/projects/` 目錄下，跨對話持續存在。

```
CLAUDE.md  → 你手動寫的，關於「專案」的靜態規則
Memory     → Claude 自己記的，關於「你這個人」的動態資訊
```

**實際運作方式：**
```
對話 1：
你：「我覺得 Optional 讓程式碼變難讀，我不喜歡用它」
Claude 記下來 → memory 裡新增一筆

對話 2（全新 session，Claude 不記得對話 1）：
你：「幫我重構這段程式碼」
Claude：（讀取 memory，知道你不喜歡 Optional）
       → 自動用 null check 寫法，不用 Optional.ofNullable
```

### Memory 的儲存位置

```
~/.claude/projects/[專案路徑的編碼]/memory/
```

每個專案有自己獨立的 memory 目錄。Memory 是純文字的 Markdown 檔案，你可以直接打開來看。

---

## 知識點：Memory vs CLAUDE.md 應該放什麼

| 資訊類型 | 放哪裡 | 原因 |
|---------|--------|------|
| 專案技術棧 | CLAUDE.md | 專案層級，所有人共用 |
| 開發規範 | CLAUDE.md | 固定不變，要版控 |
| 你個人的程式碼風格偏好 | Memory | 個人偏好，不適合 commit |
| 你目前正在做的任務背景 | Memory | 動態資訊，隨時會變 |
| 你在某個知識點的學習程度 | Memory | 個人狀態 |
| 你常犯的錯誤類型 | Memory | 個人資訊 |
| 過去討論過的設計決策 | Memory | 對話歷史的摘要 |

---

## 知識點：Memory 的幾種類型

Claude Code 的 memory 系統支援不同類型的記憶，對應不同用途：

**user（關於你這個人）**
- 你的技術背景、強項、弱項
- 你的工作習慣和偏好
- 你喜歡什麼風格的解釋

**project（關於進行中的工作）**
- 目前在做哪個功能
- 遇到什麼問題還沒解決
- 做過的設計決策和原因

**feedback（你給 Claude 的糾正）**
- 你不喜歡 Claude 做某件事
- 某種方式對你特別有效
- 你覺得 Claude 的哪些習慣要改

## 怎麼讓 Claude 記住東西

### 方法 1：直接說

```
請記住：我們的後端語言是 Java，我對 Python 不熟
請記住：我比較喜歡先看整體架構再看細節
請記住：解釋的時候用類比的方式，不要直接引用文件
```

Claude 會自動分類並儲存。

### 方法 2：在對話中自然帶出

有時候你不需要說「請記住」，Claude 會自己判斷值得記錄的資訊：

```
你：「上次那個方法我試過了，行不通，因為我們的環境不支援 JDK 21」
Claude：（自動記錄：這個環境用的不是 JDK 21）
```

### 方法 3：請 Claude 整理記憶

在一次較長的討論後，可以請 Claude 主動整理：

```
這次討論的重點請幫我記下來，
特別是我們決定用 CQRS 架構的原因。
```

## 管理你的 Memory

### 查看目前記住了什麼

```
你目前記住了哪些關於我的事？
```

或者：
```
你記得我對這個專案架構有哪些偏好嗎？
```

### 更新或修正記憶

```
請更新記憶：我現在不用 Java 了，改用 Kotlin
請修正：我其實不討厭 Optional，我只是不熟，之後可以教我怎麼用
```

### 刪除特定記憶

```
請忘掉你記的關於我 JDK 版本的事，那個資訊過時了
```

### 直接查看 memory 檔案

Memory 是儲存在本機的 Markdown 檔，你可以直接用文字編輯器打開：

```bash
ls ~/.claude/projects/
# 找到你的專案目錄
ls ~/.claude/projects/[你的專案目錄]/memory/
# 直接讀取或編輯裡面的 .md 檔
```

## 知識點：Memory 的有效範圍

- Memory 是**本機的**，不會同步到其他電腦
- Memory 是**跨對話的**，重開 Claude Code 還在
- Memory 是**按專案分開的**，不同專案的 memory 獨立
- 如果你想讓同事也有相同的背景知識 → 用 CLAUDE.md（可以 commit）

## 練習

### 步驟 1：告訴 Claude 關於你的三件事

開一個對話，告訴 Claude：
- 你的技術背景（用什麼語言、幾年經驗）
- 你在這個專案的角色或目前在做什麼
- 一個你的工作習慣或偏好

```
請記住以下關於我的資訊：
1. [你的技術背景]
2. [你目前的任務]
3. [你的某個偏好]
```

### 步驟 2：驗證跨對話記憶

關掉這個對話，重新開一個新的對話。

直接問：
```
你記得我有哪些技術背景和偏好嗎？
```

確認 Claude 能說出你在步驟 1 告訴它的事。

### 步驟 3：試著更新和刪除

```
請更新記憶：[把其中一件事改成新的版本]
```

再開新對話確認更新有效。

## 常見問題

**Q：Memory 會一直累積越來越多嗎？**
會，但 Claude Code 會自動整理和壓縮，不用擔心撐爆。建議定期問 Claude「你記得什麼」，確認記憶的準確度。

**Q：換了電腦，memory 還在嗎？**
不在。Memory 是存在本機 `~/.claude/` 的，換電腦要重新建立。重要的偏好建議也寫進 `~/.claude/CLAUDE.md`。

**Q：team member 能看到我的 memory 嗎？**
不能。Memory 只在你本機，沒有任何同步機制。

---

## 完成條件

關掉對話重新開一個新的，Claude 仍能說出你在上一個對話裡告訴它的個人資訊。

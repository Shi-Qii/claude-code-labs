# 03 — Memory（跨對話記憶）

## 你要學什麼

讓 Claude 記住你說過的事，下次開新對話不用重新解釋。

---

## Memory 是什麼

CLAUDE.md 是你手動寫的靜態說明。Memory 是 Claude 自己動態記錄的內容，存在 `~/.claude/projects/` 下面，跨對話持續存在。

```
CLAUDE.md  → 你寫的，關於專案的固定規則
Memory     → Claude 記的，關於你個人的偏好和狀態
```

**實際用途：**
```
你說：「我不喜歡用 Optional，覺得程式碼變難讀」
Claude 記下來

下次新對話：
你：幫我重構這段程式
Claude：（知道你不喜歡 Optional）自動用 null check 而不是 Optional.ofNullable
```

---

## 怎麼讓 Claude 記住東西

直接在對話裡說：

```
請記住：我們的 log 格式是 JSON，不要用字串拼接
請記住：這個專案不能用 Lombok，老闆不喜歡
請記住：我比較習慣先寫 test 再寫實作
```

Claude 會自動儲存到 memory 系統。

---

## 練習

1. 開一個新對話，告訴 Claude 關於你的三個工作習慣或偏好（例如：程式碼風格、你不喜歡的寫法、你的技術背景）

2. 關掉這個對話，重新開一個新的

3. 問 Claude：「你記得我有哪些程式設計偏好嗎？」

4. 確認它還記得

---

## 進階：主動查看 memory

你可以問 Claude：
```
你目前記住了哪些關於我的事？
```

或是叫它更新：
```
請更新記憶：我現在改用 Kotlin 了，不用 Java
```

---

## 完成條件

關掉對話重新開，Claude 還能說出你上次告訴它的偏好，不需要你重新說明。

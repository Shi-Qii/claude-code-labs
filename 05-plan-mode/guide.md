# 05 — Plan Mode + Sub-agents

## 你要學什麼

處理複雜任務時，先讓 Claude 規劃再執行，避免它亂動程式碼。以及把大任務拆給多個 agent 平行處理。

---

## Plan Mode 是什麼

一般模式下，Claude 會邊想邊改程式碼。Plan Mode 強迫它先把計畫列出來，等你確認後才動手。

**什麼時候用：**
- 要改動多個檔案的功能
- 不確定 Claude 會怎麼做
- 任何你想先看計畫再決定的時候

**怎麼進入：**

在對話裡輸入 `/plan` 或在設定裡開啟，也可以直接說：
```
先不要改程式，告訴我你打算怎麼做
```

---

## 練習 A：用 Plan Mode 設計功能

選一個你想加在專案裡的小功能（或用下面的範例），先叫 Claude 規劃：

```
我想在這個專案加一個功能：[你的功能描述]

先不要動程式碼，列出你的實作計畫：
1. 要改哪些檔案
2. 每個檔案改什麼
3. 有什麼潛在風險
```

看完計畫，確認沒問題再叫它執行。

---

## Sub-agents 是什麼

Sub-agents 讓 Claude 把工作拆給多個獨立的 agent 同時處理。

**什麼時候用：**
- 要同時查好幾個地方的程式碼
- 要平行執行多個獨立的任務
- 主要對話不想塞滿大量搜尋結果

**怎麼用：**

直接告訴 Claude：
```
用 sub-agent 同時查：
1. 這個專案有哪些 API endpoint
2. 資料庫有哪些 table
然後整合結果給我
```

---

## 練習 B：用 Sub-agent 做平行搜尋

叫 Claude 用 sub-agent 同時執行兩件事：

```
用兩個 sub-agent 同時做：
1. 找出這個專案所有有 TODO 的地方
2. 找出這個專案所有有 System.out.println 或 console.log 的地方
整合兩個結果，列成清單
```

觀察 Claude 怎麼拆分和整合結果。

---

## 完成條件

- 用 Plan Mode 看過一次 Claude 的實作計畫後再讓它執行
- 用 Sub-agent 執行過一次平行搜尋任務

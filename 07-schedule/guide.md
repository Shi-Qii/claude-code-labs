## 核心概念

### Schedule 是什麼

Schedule 讓你設定 Claude 在指定時間自動執行任務，不需要你手動觸發。任務在 **雲端**執行（Anthropic 的伺服器），所以你的電腦不需要開著。

```
Hooks  → 你開著 Claude Code 時，Claude 做某件事的前後觸發
Schedule → 不管你在不在，到了指定時間就自動執行
```

**適合的使用場景：**
- 每天早上整理昨天的 git log、產生工作摘要
- 每週五下午掃描 TODO / FIXME 清單
- 每天提醒自己 commit
- 定期檢查 CI 有沒有失敗
- 每月產生專案進度報告

---

## 怎麼建立排程

在 Claude Code 裡輸入：

```
/schedule
```

然後用自然語言描述你想做什麼、什麼時候做：

```
每天早上 9 點，讀取最近一天的 git log，
用繁體中文整理成工作摘要：
- 昨天完成了什麼
- 目前有哪些未 commit 的改動
```

Claude 會幫你設定好排程。

## 知識點：Cron 語法基本讀法

排程的時間設定用的是 cron 格式，看懂它能讓你更精確控制時間：

```
┌─── 分鐘 (0–59)
│  ┌── 小時 (0–23)
│  │  ┌─ 日 (1–31)
│  │  │  ┌ 月 (1–12)
│  │  │  │  ┌ 星期幾 (0–7，0 和 7 都是週日)
│  │  │  │  │
*  *  *  *  *
```

常用的模式：

| Cron 表示式 | 意思 |
|------------|------|
| `0 9 * * 1-5` | 週一到週五早上 9:00 |
| `0 18 * * *` | 每天下午 6:00 |
| `0 9 * * 1` | 每週一早上 9:00 |
| `0 0 1 * *` | 每月 1 號午夜 |
| `30 17 * * 5` | 每週五下午 5:30 |

你不需要自己寫 cron 語法，告訴 Claude 「每週五下午五點」，它會幫你轉換。

## 知識點：Schedule vs Hooks 怎麼選

| | Schedule | Hooks |
|---|---|---|
| 執行時機 | 指定時間，自動觸發 | Claude 使用工具時觸發 |
| 執行環境 | 雲端（你不用開著電腦） | 本機（需要 Claude Code 在執行中） |
| 適合做 | 定期報告、提醒、自動整理 | 存檔後自動 lint、自動編譯 |
| 能存取本機檔案嗎 | 能（透過 Claude Code 執行） | 能 |

## 管理你的排程

### 查看所有排程
```
/schedule list
```
或直接問：
```
我目前有哪些排程任務？
```

### 暫停排程
```
暫停「每日工作摘要」的排程
```

### 刪除排程
```
/schedule delete
```
或：
```
刪除「每日工作摘要」的排程
```

### 馬上執行一次（測試用）
```
立刻執行一次「每日工作摘要」排程，不要等到明天
```

## 實用排程範例集

### 每日工作啟動提醒

```
每個工作日早上 9:30，
讀取 TODO.md（如果沒有就建立），
顯示今天的待辦清單，
並詢問我今天的工作重點，把回答更新進 TODO.md
```

### 每週程式碼健康檢查

```
每週五下午 5:00，掃描整個專案，
找出：
1. 所有 TODO、FIXME、HACK 的數量和位置
2. 超過 200 行的函式
3. 超過 500 行的檔案
輸出一份簡短的「技術債清單」
```

### 每日 git commit 提醒

```
每天下午 6:00，
執行 git status，
如果有未 commit 的改動就提醒我記得 commit，
並列出哪些檔案改了
```

### 每週 PR 狀態整理

```
每週一早上 9:00，
呼叫 GitHub MCP 查看我的 open PR，
列出每個 PR 的狀態、最近有沒有人 review、需不需要我回應
```

（需要先安裝 GitHub MCP，參考 06 號練習）

## 練習

### 步驟 1：建立一個對你有用的排程

從上面的範例選一個，或自己設計：

```
/schedule [描述你要做什麼和什麼時候]
```

### 步驟 2：確認排程已建立

```
/schedule list
```

確認你剛才建立的排程出現在清單裡。

### 步驟 3：手動觸發一次測試

```
立刻執行一次「[你的排程名稱]」，讓我確認它運作正常
```

確認輸出符合你的預期。

## 常見問題

**Q：排程需要我的電腦開著嗎？**
不需要，Schedule 在雲端執行。但如果排程的任務需要存取你本機的檔案，Claude Code 需要在執行中。

**Q：排程的時區是哪個？**
預設是你帳號的時區設定。建立排程時可以明確說「台灣時間早上 9 點」。

**Q：排程執行失敗會通知我嗎？**
會的，Claude Code 會在你下次開啟時顯示執行結果。

**Q：可以建立幾個排程？**
根據你的方案有限制，免費方案通常可以建立幾個基本的排程。

---

## 完成條件

成功建立一個定時任務，在 `/schedule list` 裡確認它存在，並手動執行一次確認輸出正確。

# 02 · 90 分鐘工作坊腳本

**適用場景：** 公司內部 Claude Code 教育訓練，10-20 人規模。

---

## 時間軸

| 時段 | 時長 | 內容 |
|------|------|------|
| 0–10 min | 10 min | **Live demo**：講師現場用 Claude 修一個小 bug |
| 10–30 min | 20 min | **安裝 + 環境確認**：每人跑起來，完成 00-basics |
| 30–60 min | 30 min | **分組練習**：01 CLAUDE.md + 05 Plan Mode |
| 60–80 min | 20 min | **自由實作**：用自己的專案試一個真實任務 |
| 80–90 min | 10 min | **QA + 收尾** |

> **注意：** 10–30 min 的安裝段是最重要的緩衝，不要砍掉。  
> 如果參加者已事先完成安裝，這段可以現場壓縮，改成多做 00-basics 練習或提前進分組練習。  
> 但腳本預留這 20 分鐘 —— 現場永遠有人需要排錯。

---

## 分段詳細腳本

### 0–10 min｜Live Demo

**目標：** 讓大家先看到 Claude Code 真正有用的樣子，再開始操作。

選一個真實（但不敏感）的 bug，或事先準備好一個小錯誤。建議：

```
情境：Spring Boot 的一個 service method，在某個邊界條件下拋出 NPE
做法：用 @ 引用檔案，叫 Claude 找原因、修復、確認測試還是過的
```

**關鍵動作：**
1. 用 `@` 引用檔案（現場演示 @ 的選擇器）
2. 說一個有上下文的指令（不只說「修 bug」）
3. 改完後跑 `git diff` 讓大家看 Claude 改了什麼
4. 說：「這就是接下來 90 分鐘你們要學的東西」

---

### 10–30 min｜安裝 + 環境確認

**主持人說：** 「請打開終端機，我們現在確認每個人都能用。」

安裝：
```bash
npm install -g @anthropic-ai/claude-code
claude --version
```

驗證：
```bash
cd 你的任意專案資料夾
claude
```

看到 Claude Code 提示符就算成功。**讓大家完成 00-basics 的練習 A（@ 引用）**，確認能跑起來。

**排錯備案：**
- 沒有 Node.js → `brew install node` 或 `nvm install --lts`
- 公司網路擋 npm → 改用公司 mirror 或請 IT 開通
- 沒有帳號 → 需要 Claude Pro 或 Max 訂閱

---

### 30–60 min｜分組練習

分成 2–4 人一組，每組選一個：

- **01 CLAUDE.md** — 適合已有真實專案的人：讓 Claude 真正認識你的 codebase
- **05 Plan Mode** — 適合想嘗試複雜任務的人：先規劃後執行，適合重構或大改動

**主持人提示：**
- 遇到問題先看一旁的 [CHEATSHEET](../CHEATSHEET.md)
- 「做完」的定義是達到 guide.md 最後的**完成條件**，不是做完所有練習

---

### 60–80 min｜自由實作

每人用**自己的專案**，帶一個今天想處理的真實任務：

- 修一個你一直拖著沒修的 bug
- 請 Claude 幫某個複雜方法補測試
- 用 Plan Mode 規劃一個你一直覺得太複雜的重構

**主持人巡場，觀察常見問題：**
- 指令太模糊 → 提示補上上下文
- 對話太長 → 提示 `/compact`
- 沒看 git diff 就 accept → 提醒這個習慣

---

### 80–90 min｜QA + 收尾

**常見 QA：**

**Q：這個適合用在 code review 嗎？**  
可以，用 05 Plan Mode 讓它先分析再給建議，比直接叫它改更可控。

**Q：要擔心把公司 code 傳給 Anthropic 嗎？**  
詳見[04-security.md](./04-security.md)。簡短版：預設訊息用於訓練，可以在設定關閉；敏感系統建議先讀資安規範。

**Q：免費版夠用嗎？**  
基礎篇 01-07 用 Claude Pro 訂閱就夠，不額外計費。進階 API 呼叫另計。

**Q：跟 GitHub Copilot 有什麼差？**  
Copilot 主要是 IDE 內補全；Claude Code 是在終端機的對話式操作，擅長多步驟任務、跨檔案重構、解釋和規劃，用途不同。

**收尾動作：**
- 傳 repo 連結給所有人
- 建議下週 1:1 時帶一個自己試用的任務分享
